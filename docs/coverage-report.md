## Аналіз
- **Які функції/класи покриті найкраще?**
Найкраще покриті модулі в директорії `src/helper/`, зокрема файли `prepareLessonCell.js`, `shortTitle.js`, `sortArray.js` та `strings.js`. Після додавання нових тестів у межах лабораторної роботи, ці файли досягли 100% покриття за всіма показниками. Також високий показник покриття мають константи проекту.

- **Які потребують додаткових тестів?**
Додаткових тестів потребують UI-компоненти (`src/components/`) та контейнери (`src/containers/`), більшість з яких мають покриття менше 10% або 0%. Зокрема, складні модулі візуалізації розкладу (`renderScheduleTable.js`) та логіка авторизації (`Auth.js`) потребують інтеграційних тестів для перевірки взаємодії з Redux та API.

- **Чому деякі branches не покриті?**
Більшість непокритих гілок (branches) у великих файлах проекту відповідають за обробку помилкових станів (наприклад, перевірка `if (!data)` при невдалих запитах до сервера) або специфічні сценарії відображення, які не імітувалися в межах даного набору юніт-тестів. Також велика кількість гілок у React-компонентах вимагає використання інструментів для тестування DOM (наприклад, React Testing Library), тоді як поточні тести фокусувалися на чистих функціях-хелперах.
